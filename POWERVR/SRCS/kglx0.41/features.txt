V0.0 (08.25.02)
------------------------
00. support of GL_TRIANGLE_STRIP, GL_QUADS, GL_TRIANGLE_FAN, GL_POLYGON,
			   GL_QUAD_STRIP, GL_POINTS

0. used the core of KGL/KOS 1.1.9
  and gldraw.c from libgl by Bero.

1. added GL_NT_FLATFAN/GL_NT_FLATSTRIP/GL_NT_QUADS primitive typ, which is a
geometric primitive that is specified in device coordinates and is drawn
regardless of current matrix or viewport. Very useful for bypassing
the 3D->2D transformation pipeline!

WARNING: You must use  z>=0 values


2. added some more original GLU stuff

void
gluOrtho2D(GLfloat left, GLfloat right, GLfloat bottom, GLfloat top);

void
gluPickMatrix(GLfloat x, GLfloat y,
	      GLfloat width, GLfloat height, GLint viewport[4]);

GLint
gluProject(GLfloat objx, GLfloat objy, GLfloat objz,
	      const GLfloat modelMatrix[16],
	      const GLfloat projMatrix[16],
           const GLint viewport[4],
	      GLfloat *winx, GLfloat *winy, GLfloat *winz);


3.
 changed glFrustum(): does Z coord MAPPING to [1,0] not to [-1,1], cause
 the DC has a RHW-buffer NOT a Z-buffer.

  GLint glIsList(GLuint list);
  void glNewList(GLuint list, GLenum mode);
  void glEndList(void);
  void glCallList(GLuint list);

  glColor3ub

  glTexCoord2fv
