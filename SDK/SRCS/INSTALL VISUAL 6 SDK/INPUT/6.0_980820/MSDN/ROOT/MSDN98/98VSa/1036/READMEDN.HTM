Nouveautés de cette version


Nouvel utilitaire d'affichage de MSDN Library

L'édition Visual Studio 6.0 de MSDN Library est la première version de la bibliothèque MSDN Library qui utilise la technologie de l'utilitaire d'affichage de l'aide HTML. Pour obtenir de l'aide sur l'utilitaire d'affichage, cliquez sur Aide dans la barre de menus de MSDN Library puis sur Aide MSDN Library.


Problèmes liés à l'installation


Informations d'ordre général sur l'installation

Vous ne devez pas nécessairement désinstaller les versions antérieures de MSDN Library qui utilisent la technologie InfoViewer 5.0. En effet, la bibliothèque de juillet 1998 et toutes les bibliothèques précédentes coexistent dans l'édition Visual Studio 6.0 de MSDN Library. Si vous souhaitez toutefois désinstaller une bibliothèque, utilisez l'option Ajout/Suppression de programmes du Panneau de configuration ou du programme d'installation du CD-ROM de MSDN Library correspondant à la version à désinstaller.


Options d'installation Standard, Complète et Personnalisée

Lors de l'installation, vous serez amené à choisir entre les options d'installation Standard, Complète et Personnalisée. Sachez que quelle que soit l'option d'installation sélectionnée, vous pourrez accéder à l'intégralité de la documentation et des exemples. Les options d'installation vous permettent de déplacer la plupart des fichiers sur votre disque dur, ce qui vous permet d'être moins dépendant du CD-ROM MSDN Library.

L'option d'installation Standard vous permet d'exécuter la bibliothèque MSDN Library à partir du CD-ROM. Le programme d'installation copie ainsi un minimum de fichiers sur votre disque dur, comme les fichiers nécessaires à l'utilitaire d'affichage de MSDN Library, les fichiers d'index du sommaire et les fichiers d'aide utilisés par les produits de développement Visual Studio. Vous pouvez également afficher la table des matières de MSDN Library. Si vous sélectionnez des éléments de la table des matières qui n'ont pas été copiés sur votre disque dur, un message s'affiche, vous invitant à insérer le CD-ROM MSDN Library.

L'option d'installation Complète vous permet d'exécuter la bibliothèque MSDN Library presque exclusivement à partir de votre disque dur. Le programme d'installation copie l'intégralité de la documentation de MSDN Library ainsi que certains exemples de fichiers sur votre dique dur. (Sachez cependant que les exemples de Visual C++â, de MSDN et de Visual Modeler ne sont pas copiés sur votre disque dur lors d'une installation complète.) Les fichiers copiés incluent également les fichiers nécessaires à l'utilitaire d'affichage de MSDN Library, les fichiers d'index du sommaire et les fichiers d'aide utilisés par les produits de développement Visual Studio.

L'option d'installation Personnalisée vous permet de choisir une installation locale d'un sous-ensemble de la bibliothèque MSDN Library. Les fichiers que vous sélectionnez sont copiés sur votre disque dur avec les fichiers mentionnés dans la description de l'installation Standard. Vous pouvez accéder à la documentation du produit sans avoir à utiliser le CD-ROM MSDN Library pour les fichiers sélectionnés en vue d'une copie sur votre disque dur. Outre ces produits et ces exemples de fichiers, sélectionnez également l'index de recherche sur le texte entier qui vous permet d'effectuer des recherches sur la documentation de votre produit sans avoir à insérer le CD-ROM de la bibliothèque MSDN Library. Vous pouvez également afficher l'intégralité de la table des matières de MSDN Library. Si vous sélectionnez des éléments de la table des matières qui n'ont pas été copiés sur votre disque dur, un message s'affiche, vous invitant à insérer le CD-ROM MSDN Library.


Installation de MSDN Library sur un serveur réseau

Pour installer la bibliothèque MSDN Library sur un serveur réseau, copiez le CD-ROM nº1 et le CD-ROM nº2 sur un répertoire commun du réseau.

  1. Créez un répertoire sur votre lecteur réseau, par exemple, C:\MSDN VS6.
  2. Créez les sous-répertoires suivants : C:\MSDN VS6\CD1 et C:\MSDN VS6\CD2.
  3. Copiez le contenu des CD-ROM nº1 et nº2 dans leur répertoire respectif sous C:\MSDN VS6.
  4. Copiez le fichier Msdn332.inf depuis C:\MSDN VS6\CD2 vers C:\MSDN VS6\CD1.
  5. Copiez le contenu de C:\MSDN VS6\CD2\MSDN vers C:\MSDN VS6\CD1\MSDN.
  6. Cette opération vous permet de supprimer le contenu du répertoire C:\MSDN VS6\CD2.
  7. Supprimez le répertoire C:\MSDN VS6\CD2.
  8. Double-cliquez sur le programme d'installation setup.exe sous C:\MSDN VS6\CD1 pour installer la bibliothèque MSDN Library sur le réseau.


Message « Vérification de la version d'Internet Explorer » lors de l'installation

Pour que MSDN Library fonctionne correctement, vous devez disposer de la version 4.01 de Microsoft Internet Explorer, avec le Service Pack 1, ou d'une version ultérieure installée sur votre système. Si vous n'installez pas la version recommandée de Internet Explorer (Internet Explorer 4.01 avec le Service Pack 1), certaines fonctions de MSDN ne seront pas disponibles. Pour de plus amples informations, reportez-vous à la rubrique Limites de Internet Explorer 3.02. Rappelez-vous que la version 3.02 de Internet Explorer ou une version ultérieure est requise.

Lors de l'installation, le message « Vérification de la version d'Internet Explorer » peut apparaître si vous installez MSDN Library sans suivre la procédure d'installation recommandée de Visual Studio. Ce message interactif s'affiche si vous utilisez une version de Internet Explorer antérieure à la version fournie avec le produit Visual Studio 6.0. L'intitulé partiel de ce message est le suivant : « Choisissez Oui pour arrêter l'installation et installer une version ultérieure d'Internet Explorer. Choisissez Non pour poursuivre l'installation. »

Si vous cliquez sur Oui, le programmme d'installation de MSDN Library s'interrompt et le message suivant  s'affiche : « MSDN Library - Installation de Visual Studio 6.0: l'opération n'a pas pu être terminée. » Vous devez cliquer sur OK, puis installer Internet Explorer manuellement. La version 4.01 de Internet Explorer est incluse sur le CD-ROM de Visual Studio. Il vous suffit d'exécuter le fichier ie4setup.exe pour l'installer. Vous pouvez également vous connecter au site http://www.microsoft.com/ie_intl/fr/ pour télécharger la dernière version de Internet Explorer. (Remarque : Si vous utilisez Windows NT 4.0, vous devrez peut-être installer le Service Pack 3. Pour de plus amples informations, veuillez consulter le site http://www.microsoft.com/ie_intl/fr/). Si vous avez choisi d'installer une version ultérieure de Internet Explorer, vous devez redémarrer le programme d'installation de MSDN Library et poursuivre l'installation.

Si vous cliquez sur Non, l'installation se poursuit.

Si vous utilisez Internet Explorer 3.02 sous Windows 95, vous devez disposer de DCOM 95 ou d'une version ultérieure pour que MSDN Library fonctionne. DCOM 98 est intégré dans le programme d'installation standard de Visual Studio.


Impossible de trouver le fichier de collection lors de l'installation avec Windows Terminal Server

L'édition Visual Studio 6.0 de MSDN Library ne fonctionne pas correctement si vous utilisez Windows Terminal Server. Une fois la bibliothèque MSDN Library installée, si vous sélectionnez Démarrage, Microsoft Developer Network, puis MSDN Library Visual Studio 6.0, un message d'erreur s'affiche, indiquant que le fichier msdn*.col n'a pas pu être trouvé.

Solution

Pour que MSDN Library fonctionne correctement, déplacez le fichier hhcolreg.dat du répertoire Windows\Aide vers le répertoire d'aide du profil d'utilisateur correspondant. Le répertoire du profil d'utilisateur se trouve dans le répertoire Windows\Profils.


Message de collection par défaut lors de l'installation

Un message peut s'afficher au terme de l'installation de MSDN Library vous demandant si vous souhaitez que le programme d'installation de MSDN Library définisse l'édition Visual Studio 6.0 de MSDN Library en cours d'installation comme collection par défaut du shell développeur. Si vous cliquez sur Oui, les fichiers d'aide des produits Microsoft Developer, obtenus via la touche F1, sont transférés vers le répertoire d'installation de la nouvelle version de MSDN Library. Si vous cliquez sur Non, les fichiers d'aide des produits Microsoft Developer, obtenus via la touche F1, sont transférés vers le précédent fichier de collection par défaut.

Pour de plus amples informations sur la modification des collections, reportez-vous à la rubrique Collections — Modification de la collection par défaut.


Désinstallation de cette version

Pour supprimer MSDN Library de votre machine, cliquez sur Ajout/Suppression de programmes dans le Panneau de configuration. Sélectionnez MSDN Library - Visual Studio 6.0 (français) puis cliquez sur Ajouter/Supprimer. Cliquez sur le bouton Supprimer tout.


Problèmes connus


Collections — Modification de la collection par défaut

L'édition Visual Studio 6.0 de MSDN Library constitue la référence en matière de documentation et d'aide pour tous les produits Visual Studio. Lors de l'installation de Visual Studio, l'édition Visual Studio 6.0 de MSDN Library est définie comme la collection de documents par défaut pour l'aide obtenue via la touche F1 et l'aide générale sous forme de documentation. Si une autre collection de documents du développeur est installé, vous pouvez désigner cette collection comme la collection par défaut pour l'aide obtenue via la touche F1 et l'aide générale sous forme de documentation. Vous pouvez modifier la collection par défaut dans l'environnement de développement intégré (IDE) de Visual J++ä/Visual InterDevä ou dans Visual C++ en sélectionnant Outils, Options, puis Système d'aide.

Vous devez toujours fermer la fenêtre de MSDN Library avant de modifier la collection de documents par défaut de Visual Studio. Dans le cas contraire, vos modifications ne seront pas enregistrées.


Limites de Internet Explorer 3.02

Le sommaire ne s'affiche pas dans le volet droit de l'utilitaire d'affichage de MSDN Library

Lorsque vous utilisez Internet Explorer 3.02, si le Gestionnaire d'accès est activé, le contenu n'ayant pas été évalué, comme le sommaire de MSDN Library, ne s'affiche pas. Pour afficher le sommaire de MSDN Library, sélectionnez Options dans le menu Affichage. Cliquez sur l'onglet Sécurité, puis sur l'option de désactivation du contrôle d'accès.

Remarque : Lorsque vous modifiez les options Internet pour MSDN Library, ces options sont également modifiées pour Internet Explorer.

Le texte sélectionné ne s'imprime pas

Avec Internet Explorer 3.02, il n'est pas possible d'imprimer le texte sélectionné dans une page de MSDN Library.

Le texte n'est pas mis en surbrillance après une recherche sur le texte entier

Avec Internet Explorer 3.02, il n'est pas possible de mettre en surbrillance les résultats d'une recherche sur le texte entier.


Problèmes liés à Internet Explorer 4.01

Impossible de couper et de coller correctement les exemples de code

Les ruptures de ligne et les informations de format ne sont pas nécessairement copiées correctement lorsque vous copiez, puis collez les exemples de code de la documentation Visual Studio de MSDN Library dans votre éditeur de code. Pour remédiez à ce problème, procédez de l'une des manières suivantes :

  • Éditez manuellement les ruptures de lignes une fois que vous avez copié le code.

  • Affichez la source de l'exemple de code, puis copiez l'intégralité de l'exemple de code, notamment les balises <pre> et </pre>. Copiez-le dans votre éditeur de code, puis supprimez les sections inutiles de la version qui a été collée.

Rupture des liens de texte et messages d'avertissement de la sécurité des liens

Si les paramètres de sécurité sont définis pour désactiver le scripting et les contrôles ActiveXâ, certains liens de MSDN ne fonctionnent pas. Pour modifier les paramètres relatifs à la sécurité, sélectionnez Options Internet dans le menu Affichage. Cliquez sur l'onglet Sécurité, puis sélectionnez la zone appropriée (la Zone Intranet s'applique à la plupart des utilisateurs). Définissez un niveau de sécurité moins strict pour que les liens fonctionnent.

Remarque : Lorsque vous modifiez les options Internet pour MSDN Library, ces options sont également modifiées pour Internet Explorer.


Index - Les sous-entrées d'index ne sont pas triées sous les entrées principales correspondantes

Si deux mots clés diffèrent uniquement de par leur casse et qu'au moins un des mots clés contient des sous-entrées d'index, les sous-entrées des deux mots clés s'affichent sous le mot-clé qui a été trié en dernier lieu dans l'index.

Par exemple, les mots clés suivants sont traités comme suit :
   liaison de données
      contrôles ActiveX
   Liaison de données
      concepts clé

Ces mots clés apparaitront comme suit dans l'index de MSDN Library :
   liaison de données
   Liaison de données
      contrôles ActiveX
      concepts clé

Liaison de données a été trié dans l'index après liaison de données. Liaison de données contient, par conséquent, les sous-entrées d'index des deux mots clés.


Le bouton Localiser est activé mais ne fonctionne pas

Pour certaines rubriques, particulièrement celles trouvées dans la documentation de Visual Basicâ, le bouton Localiser ne fonctionne pas, même s'il est activé.


Disparition des flèches de navigation

Certaines rubriques dans MSDN Library sont volontairement « masquées » dans la table des matières. Ces rubriques désactivent les flèches de navigation. Vous devez cliquer sur une nouvelle rubrique dans la table des matières pour passer aux rubriques précédente ou suivante.


Impression - Impression de plusieurs rubriques

Nous vous conseillons de ne pas imprimer plusieurs rubriques. Si vous imprimez une rubrique volumineuse, une importante quantité de mémoire est utilisée et l'impression prend du temps. Si la quantité de mémoire disponible sur votre machine est trop faible, un message vous indique que la mémoire n'est pas suffisante.


Impossible d'imprimer le contenu de plusieurs cadres simultanément

Si vous essayez d'imprimer une rubrique de la table des matières contenant d'autres rubriques avec un contenu HTML basé sur les « cadres », le message suivant s'affiche : « Les rubriques sélectionnées doivent être imprimées individuellement ». Vous devez alors sélectionner un livre ou une page à un niveau inférieur pour être en mesure d'imprimer la rubrique de votre choix.


Problèmes liés aux sous-ensembles

Utilisation des sous-ensembles pour limiter une recherche effectuée sur le texte entier

Pour de plus amples informations sur la création de sous-ensembles, reportez-vous à l'Aide MSDN Library (sélectionnez le menu Aide, puis cliquez sur Aide MSDN Library). Ce document ne contient pas les informations suivantes relatives à l'utilisation des sous-ensembles pour limiter une recherche sur le texte entier :

  1. Dans le volet gauche de la documentation en ligne, cliquez sur l'onglet Recherche.
  2. Dans la liste Sous-ensemble actif, cliquez sur le nom du sous-ensemble que vous recherchez.
  3. Cliquez sur Liste des rubriques pour lancer la recherche.

La page Rubrique non trouvée n'inclut pas les sous-ensembles

Lorsque vous utilisez les sous-ensembles et que vous essayez de suivre un lien d'une rubrique qui mène à une rubrique cible en dehors de l'ensemble actif, la page « Rubrique non trouvée » apparaît. Cette page ne vous indique pas que la rubrique n'a pas été trouvée car elle existe en dehors du sous-ensemble courant. Sélectionnez le sous-ensemble Toute la collection pour accéder à cette rubrique.

La liste de restriction est ignorée

Lorsque vous utilisez les sous-ensembles, les recherches sur le texte entier n'excluent pas les mots de la liste de restriction de MSDN Library.

La fonctionnalité Rechercher les résultats précédents ne fonctionne pas

Lorsque vous utilisez les sous-ensembles, la fonctionnalité « Rechercher les résultats précédents » de la recherche sur le texte entier ne fonctionne pas.

Suppression des sous-ensembles définis par l'utilisateur

Dans certains cas, si vous fermez puis ouvrez la bibliothèque MSDN Library rapidement, certaines informations définies par l'utilisateur peuvent être perdues, notamment les sous-ensembles définis par l'utilisateur et récemment créés. Pour éviter de perdre ces informations, veuillez attendre que MSDN Library soit fermée avant de redémarrer MSDN Library.


Renvoi du message « Aucune rubrique trouvée » lors de la recherche

Après une installation standard, si vous effectuez une recherche sur un document alors que le deuxième CD-ROM n'est pas inséré dans le lecteur, puis vous cliquez sur le bouton Annuler, le message « Aucune rubrique trouvée » s'affiche. Si vous effectuez ensuite une recherche lors de la même session, le message « Aucune rubrique trouvée » s'affiche au lieu du message vous demandant d'insérer le deuxième CD-ROM.

Solution

Insérez le deuxième CD-ROM lorsque le message correspondant s'affiche puis cliquez sur OK. Dans ce cas, la recherche s'effectuera correctement. Vous pouvez également quitter la session afin de l'annuler, puis redémarrer MSDN Library.


Les résultats des recherches peuvent faire apparaître des caractères étrangers

Dans certaines langues, essentiellement celles qui utilisent un script bidirectionnel, les résultats des recherches peuvent contenir des caractères étrangers ou n'apparaissant pas dans l'ordre. Ce problème peut survenir lorsqu'une recherche contient des caractères génériques, des caractères spéciaux ou des marques de ponctuation. Les résultats des recherches fonctionnent correctement, seul l'affichage n'est pas correct.


Connexion impossible avec les URL indiquées

En raison de la nature changeante du Web, il est possible que certaines des URL (Uniform Resource Locators) de connexion mentionnées au moment de la publication de cette édition ne soient plus valables. Sachez cependant que MSDN Library fait de son mieux pour garder les URL à jour. Si vous souhaitez nous faire connaître un problème lié à un lien, envoyez-nous un message électronique rédigé en anglais et dans lequel vous indiquez l'emplacement du lien posant problème, à l'adresse suivante : msdn@microsoft.com.


Modification des numéros de téléphone

Il est possible que certains numéros de téléphone mentionnés dans la documentation d'accompagnement de la bibliothèque MSDN Library ne soient plus valables. Sachez cependant que MSDN Library fait de son mieux pour garder les numéros de téléphone à jour.


Case à cocher « Regrouper les mots identiques » dans l'onglet Recherche

Si vous activez cette case à cocher, MSDN Library effectue alors une recherche sur les variantes du terme entré dans la requête de recherche, sauf si celui-ci se trouve entre guillemets. Ainsi, si vous recherchez le terme debug, les termes de la même racine que debug, tels que debugged et debugging, seront renvoyés au terme de la recherche. Sachez toutefois que cette fonctionnalité a été spécialement conçue pour être utilisée dans du texte anglais et que les résultats obtenus dans votre langue risquent d'être erronés. MSDN Library vous conseille d'activer cette case à cocher lors de recherches sur des termes en anglais et de la désactiver lors de recherches dans votre langue.


Informations d'ordre général


À quoi correspondent les fichiers COL, CHM, CHI, CHW et CHQ sur mon disque dur et sur le CD-ROM ?

Les fichiers de document de MSDN Library sont des fichiers HTML compilés portant l'extension .CHM. Les fichiers CHI, CHW et CHQ correspondent à des fichiers de l'index, créés afin d'améliorer les performances de la fonction de recherche d'index, de mots clés et de recherche sur le texte entier de l'utilitaire d'affichage de MSDN Library.

Le fichier .COL définit la table des matières d'un ensemble de documents spécifiques, constitués de fichiers CHM.


Recherche d'informations sur les balises HTML

Les informations sur les balises HTML ont été déplacées. Étant donné qu'elles ne sont pas incluses dans l'index des mots clés, elles ne sont pas accessibles via la touche F1. Pour de plus amples informations sur les balises HTML, cliquez sur l'onglet Sommaire dans le volet de navigation de la bibliothèque MSDN Library pour ouvrir la table des matières. Vous trouverez également des informations sur les éléments HTML dans le dossier du SDK Platform :

 » SDK Platform
  » Services Internet, Intranet et Extranet
  » Dynamic HTML
  » DHTML References
  » HTML References
  » HTML Elements


Sommaire de MSDN Library

Le sommaire de la bibliothèque MSDN Library est en constante évolution. Nous vous avons fourni la documentation sur le produit ainsi qu'une partie de la bibliothèque MSDN Library actuelle.

L'équipe Microsoft Developer Network n'apporte généralement pas de corrections aux spécifications, aux articles et à la documentation sur le produit inclus dans MSDN Library. Aussi, il est possible que vous rencontriez des erreurs dans la documentation ou des problèmes de formatage et de référence à des numéros de page. Nous communiquerons toutes les erreurs que vous nous reporterez aux groupes responsables de la documentation sur le produit. Si vous souhaitez nous faire connaître un problème éventuel, envoyez-nous un message électronique, rédigé en anglais, à l'adresse suivante : msdn@microsoft.com.


Contenu des CD-ROM

La bibliothèque MSDN Library est incluse sur deux CD-ROM. La documentation, les exemples et les fichiers d'installation ont été placés sur les CD-ROM pour utiliser au mieux l'espace disponible et limiter les changements de CD-ROM. Le CD-ROM nº 1 contient les exemples de fichiers et les fichiers d'installation tandis que le CD-ROM nº 2 contient les fichiers de document. Trois fichiers de document sont situés sur le CD-ROM nº 1 : deux fichiers .CHM (dsmsdn.chm et dshelp.chm) installés sur votre disque dur quelle que soit le type d'installation sélectionné, et un troisième fichier .CHM (vcsample.chm) utilisé pour relier les exemples sur le CD-ROM nº 1.


Exemples décompressés

Les exemples, inclus sur le CD-ROM nº 1 sous forme décompressée, sont disponibles pour chacun des produits Visual Studio et se situent dans leur répertoire respectif. Certains exemples sont accessibles par le biais de liens à des documents. D'autres sont référencés dans la documentation du produit Visual Studio de la bibliothèque MSDN Library.


Liste de restriction de MSDN Library

Une liste de restriction de termes a été utilisée afin de réduire la taille de l'index des mots clés et d'améliorer les performances de la fonction de recherche. La liste de restriction de la bibliothèque MSDN Library  contient des termes suivants : a, am, an, and, are, as, at, be, been, being, but, by, can, could, did, do, does, doing, done, for, from, had, has, have, he, here, how, I, in, is, it, its, may, not, of, on, or, saw, see, seen, she, so, that, the, them, then, there, these, they, this, those, to, too, very, was, we, when, where, which, with, you.

Si vous entrez une requête de recherche qui utilise un terme de la liste de restriction, le message d'erreur suivant s'affiche : « La syntaxe de la requête de recherche est incorrecte. »
