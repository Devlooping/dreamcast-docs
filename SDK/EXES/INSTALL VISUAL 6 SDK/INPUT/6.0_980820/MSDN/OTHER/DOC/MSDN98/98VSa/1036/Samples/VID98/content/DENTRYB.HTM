L'envers du décor

Cet exemple utilise un seul fichier : Entry.asp. Ce fichier utilise plusieurs contrôles DTC différents pour parcourir et éditer les enregistrements de la base de données et y naviguer.

Recordset DTC : Ce contrôle connecte la page à la table DENTRY.

TextboxDTC : Ces contrôles permettent d'afficher et d'éditer les champs Prénom, Nom, Adresse, Ville, Pays/Province et Code postal de la table DENTRY. Chacun de ces contrôles est lié à un champ de la table.

OptionGroup DTC : Ce contrôle est lié au champ SEXE de la table DENTRY. Les valeurs statiques de Masculin et Féminin sont définies dans les propriétés du contrôle. Elles correspondent à 1 pour Masculin et à 0 pour Féminin dans la table.

Listbox DTC: Ce contrôle est défini comme zone de liste déroulante et est lié au champ REVENUS de la table DENTRY.

Checkbox DTC : Ce contrôle est lié au champ PC de la table DENTRY. La valeur du champ est une valeur binaire qui détermine si la case à cocher doit être activée ou pas.

RecordsetNavBar DTC : Ce contrôle est lié au contrôle DTC Recordset pour permettre à l'utilisateur d'exécuter pas à pas les enregistrements de la table DENTRY.

FormManager DTC : Ce contrôle gère le formulaire tout entier. Lorsque le formulaire est affiché la première fois, il est en mode Navigation. Deux boutons visibles en haut du formulaire permettent d'alterner entre les modes Navigation et édition. Lorsque vous passez en mode édition, trois nouveaux boutons deviennent visibles et le contrôle RecordsetNavBar est masqué. Les trois boutons supplémentaires permettent d'Insérer, Supprimer ou Actualiser des informations dans la table DENTRY.
Dans la boîte de dialogue Propriétés de FormManager, les propriétés des modes Navigation et édition sont définies de façon à permettre d'activer et de désactiver les contrôles du formulaire. Le mode Navigation est également défini pour être le mode par défaut quand vous accédez à cette page Web.
Les propriétés des boutons sont définies dans l'onglet Action de la boîte de dialogue Propriétés de FormManager. Chaque bouton exécute un ensemble différent d'actions.

Le bouton éditer permet d'activer les contrôles du formulaire, d'afficher trois boutons pour Insérer, Supprimer et Actualiser. Il permet aussi de masquer le contrôle RecordsetNavBar.

Le bouton Insérer permet d'appeler la méthode Recordset.AddRecord puis la méthode Recordset.MoveLast afin de vous permettre d'éditer l'enregistrement nouvellement ajouté.

Le bouton Supprimer permet d'appeler la méthode Recordset.DeleteRecord afin de supprimer l'enregistrement sélectionné puis d'appeler la méthode Recordset.MoveFirst pour afficher le premier enregistrement du jeu d'enregistrements.

Le bouton Actualiser appelle la méthode Recordset.UpdateRecord afin d'enregistrer les modifications apportées à l'enregistrement sélectionné. Il faut cliquer sur ce bouton pour enregistrer les modifications apportées à un enregistrement créé à l'aide du bouton Insérer.

Le bouton Navigation permet de désactiver les contrôles du formulaire, de masquer les trois boutons d'édition et d'afficher le contrôle RecordsetNavBar.
