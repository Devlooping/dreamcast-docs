Known Problems/Issues

The following is a list of known problems with WndTabs. Some may be corrected in future versions, but all are nevertheless harmless:

  1. DevStudio-initiated changes in toolbar location (for example, due to the appearance of the "Dialog" toolbar) may cause the window tabs to disappear on rare occasions.
    A workaround for this problem is to force a change in the size of the editor window (for instance, by opening and closing the Output pane).


  2. On some systems, the "Reset Menus" button in the Options|Menus dialog has effect only after DevStudio is restarted.


  3. Programs listed under the "Open With..." menu (in the shell context menu) display the Open With... dialog instead of executing.


  4. Under some circumstances, when using the persistent tabs option (Options|Tab Ordering), tab positions aren't saved properly or not at all.


  5. WndTabs popup menus sometimes don't disappear until you click somewhere in the Visual C++ window.


  6. On some systems, resource tabs aren't grouped, even when the "From same .rc file" option is selected in the Options|Tabs|Advanced dialog.


  7. The Visual C++ "Recycle Windows" option isn't fully supported.  Under some conditions, tabs won't properly update when a window is cycled.  A workaround is to use the auto tab refresh feature (in Options|Tabs).


  8. Under rare conditions, tabs get totally out of sync with the open windows.  Closing all windows fixes this problem.


You should also check out the information about compatibility with other add-ins and applications.  Also, the online FAQ and support page will always have the most up to date information about common problems and solutions.
