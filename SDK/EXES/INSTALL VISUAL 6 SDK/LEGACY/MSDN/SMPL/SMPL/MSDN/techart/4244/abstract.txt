 
 
 PicCube: Demonstrates OpenGL Screen Saver with Texture
 Mapping
 
 The PicCube sample screen saver demonstrates how to
 texture-map OpenGL(TM) objects with Microsoft(R)
 Windows(R) device-independent bitmaps (DIBs).
 
 The "OpenGL VII: Scratching the Surface of Texture
 Mapping"msdn_gl7 technical article in the Microsoft
 Development Library discusses the texture-mapping
 techniques illustrated by PicCube. The "Creating 32-Bit
 Screen Savers with Visual C++ and MFC"msdn_scrnsave
 technical article describes how to write screen savers
 with the Microsoft Foundation Class Library (MFC).
 
 To use PicCube as a screen saver, copy the PICCUBE.SCR
 file to your Microsoft Windows NT(TM) /SYSTEM32 directory.
 Use the Control Panel to select the file as the screen
 saver, and use the Setup dialog box to pick the bitmaps
 for texturing the sides of the cube.
 
 PicCube uses a static-link version of GLlib, which
 contains a class library for OpenGL. Make sure that when
 you build PicCube, GLlib, and Nigel Thompson's animation
 libraries, you link statically to MFC instead of linking
 to the shared library. To facilitate installation, PicCube
 should use static-link versions of GLlib and MFC.
 
 PicCube was built with Microsoft Visual C++(TM) version
 2.1 and MFC version 3.1, and tested under Microsoft
 Windows NT version 3.5 on the Intel(R), MIPS(R), and
 DEC(R) Alpha AXP(TM) platforms.
