 
 
 HOUSE & APPLIANCES: Demonstrate Using COM Objects from
 Visual Basic
 
 The 32-bit VBHOUSE sample application and its associated
 APPLIANCES.DLL accompany the "MFC/COM Objects 6: Using COM
 Objects from Visual Basic"msdn_house6 technical article in
 the Microsoft(R) Development Library. This sample shows a
 simple Component Object Model (COM) object used by an
 application built with Microsoft Visual Basic(R) version
 4.0. The COM object was built with Microsoft Visual
 C++(TM) version 2.1, and tested under Microsoft Windows
 NT(TM).
 
 Note that to run the VBHOUSE sample, you must first
 register the COM objects in the registry. Copy VBHOUSE.EXE
 and APPLIANCES.DLL to your local disk, then edit
 APPLIANCES.REG to show the path to where you installed
 APPLIANCES.DLL. Type "REGEDIT -s APPLIANCES.REG" (without
 the quotation marks) in a command window to create the
 registry entries.
 
 This sample was built with Microsoft Visual Basic version
 4.0, and tested under Microsoft Windows NT.
 
 Please also note that the version of Visual Basic 4.0 used
 to build the sample is an internal beta. This sample is
 provided as an illustration of what should be possible
 when Visual Basic 4.0 ships.
